# Django Backend
This is the backend of the project, built using Django. It serves as the API for the frontend and handles all the business logic and database interactions.

## Run

Install the required packages:
```bash
pip install -r requirements.txt
```

Migrate the database and create adminuser and guest users:
```bash
python manage.py db_init
```

Run the Django server:
```bash
python manage.py runserver
```

# System Overview

```mermaid
flowchart TB
    A["User input: files
      -------------------
      Flights (CSV, JSON)
      Aircrafts (CSV, JSON)
      Maintenances (CSV, JSON)
      Optional:
      Metadata (JSON)
      Parameters (JSON)
      Rules Matrix (CSV)
      Input file (JSON)"] -->
    AB["Parsing
      ------------------
      The system will guess the data from file formats: CSV, JSON and parse it
      into internal data structures."]
    AB --> B["Individual validation"]
    B --- BA["Required fields
             ------------------------
             All required fields must be present in:
             Flights
             Aircrafts
             Maintenances"]
    B --- BB["Reasonable values
             ------------------------
             flight start before end
             turn times positive
             period start before end
             "]
    B --- BC["IDs
             ------------------------
             All flights, aircraft and maintenances must have unique IDs, either
             explicityly given or autogenerated."]

    BA --> E["Relational validation"]
    BB --> E
    BC --> E

    E --- EA["Cross-check
             ----------------
             All maintenances must reference an existing aircraft ID.
             Flights can be pre-assigned to aircraft by referencing an existing aircraft ID"]

    EA --> F["User input - Parameters
             ---------------------------------
             The user can either choose a set of predefined parameters, or customize them here."]
    F --> G["Optimization pre-process"]
    G --- GA["Set period_start and period_end
             ----------------------
             Priority order:
             1. period_start and period_end fields are set in the metadata from
             any of the input files or through user input
             2. If only one of period_start or period_end is given in the
             metadata the values will be derived from the first and last
             flight"]
    G --- GB["Set min turn time for aircraft
             -------------------------
             Priority order:
             1. min_turn_time field is set on the aircraft from any of the input files
             2. custom min turn time from parameters (matched against aircraft type field)
             3. default min turn time from parameters"]
    G --- GC["Set last_known_station
             -------------------------
             Priority order:
             1. last_known_station field is set on the aircraft from any of the input files
             2. Derive it from the last flight of the aircraft before
             period_start (requires flights to be pre-assigned to aircraft)
             "]
    G --- GD["Set available_from for aircraft
             -------------------------
             Priority order:
             1. available_from field is set on the aircraft from any of the input files
             2. Derive from the last flight of the aircraft before period_start (requires flights to be pre-assigned to aircraft)
             "]
    GA --> H["Input file for optimization validation
             ---------------------------
             One last validation of the final input to be sent to the optimizer"]
    GB --> H
    GC --> H
    GD --> H
    H --> I["Send to optimizer"]
    I --> J["Receive results from optimizer"]

```
